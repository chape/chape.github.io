微信支付

最近产品需求在App中添加微信支付功能，网上也有不少资料，但都太杂了，这里将服务端具体实现流程与踩的坑记录下来，便于以后查阅^_^ 前言 参考文献: 微信支付开发文档 微信公众平台支付接口调试工具 GitHub WxPay参考库 利用SSH反向通道在本地调试微信公共号 开始： 微信支付交互时序图 根据官方提供的时序图，我们需要明确后端需要提供三个接口: 预下单接口（与客户端交互） 微信回调接口（与微信服务器交互） 支付查询接口（与客户端交互） 预下单接口 比如我们可以命名为：http://xxx.com/weixinpay/preOrder headerParam 用户ip headerParam 用户access_token formParam 充值商品id 业务步骤： 生成本地系统订单号out_trade_no 组装待提交给微信统一下单API所需的数据(微信应用号app_id,商户号mch_id,微信回调notify_url,商品详情body,本地商户订单号out_trade_no,支付金额total_fee,用户ip spbill_create_ip,支付类型trade_type NATIVE/APP/JSAPI/WAP)，并根据前面的数据生成签名sign,最后组装为XML提交给微信统一下单API 请求微信统一下单接口并组织返回结果数据(XML) 根据结果数据组织客户端协商定义的数据格式，并序列化传给客户端(先检查数据结果中签名与本地签名是否一致，根据微信应用号app_id,时间戳time_stamp,随机字符串nonce_str,预付订单号prepay_id,签名方式sign_type,产生签名sign，最后序列化到客户端) 微信回调接口 用户支付后，微信服务器将调用此接口notify_url,本接口只与微信交互，consumes和produces均使用application/xml,本人使用org.jboss.resteasy:resteasy-jaxb-provider:2.2.0.GA将XML数据直接转换为javabean。 比如我们可以命名为：http://xxx.com/weixinpay/callback 业务步骤： 根据微信传递的请求数据生成签名，并校验来自微信的签名 根据微信回调支付结果修改本地订单号out_trade_no状态 返回微信服务器xml信息 查询订单接口 客户端传递本地订单号outTradeNo或微信订单号transaction_id查询订单状态 比如我们可以命名为：http://xxx.com/weixinpay/order queryParam out_trade_no queryParam 微信订单流水号transaction_id 业务步骤： 获取客户端订单号，根据微信app_id,商户mch_id,本地订单号outTradeNo或微信订单号transaction_id,随机字符串nonce_str,计算签名sign POST请求，提交第一步的XML数据至微信查询订单API 获取响应数据，校验签名，并比对本地订单状态，返回客户端订单状态 注意事项 字段需按照字母顺序，除去sign字段，并且最后加上key=API_KEY使用MD5签名,可以比对微信公众平台支付接口调试工具验证签名正确性。 本地机器做回调测试时，一般没有公网ip，可以使用在公网机器反向代理到本地ssh -R :[10001]:[localhost]:[8001] root@[公网ip] -i ~/.ssh/pub_rsa,其它测试方式可参考利用SSH反向通道在本地调试微信公共号 微信接口： 微信统一下单APIhttps://api.mch.weixin.qq.com/pay/unifiedorder 微信查询订单APIhttps://api.mch.weixin.qq.com/pay/orderquery

记录我的博客

这是我的第一篇博客,先尿个尿标记下如何使用git pages创建个人博客吧，^_^ 准备 操作环境: mac OSX 工具： iTerm git Hugo brew 账户： github账户 开始： 安装Hugo 使用brew安装hugo,brew install hugo 初始化本地站点 使用hugo命令hugo new site YOUR_LOCAL_DIR,运行后YOUR_LOCAL_DIR下会产生如下目录结构： archetypes content layouts static config.toml 目录解析 archetypes目录还母鸡这个是用来干嘛的，和maven脚手架应该差不多，简化创建过程用的吧。content目录存放markdown文章，layouts目录存放网站的布局模板文件，static目录里放的是一些图片、css、js等资源。进入YOUR_LOCAL_DIR目录,cd YOUR_LOCAL_DIR，创建博客MD文件，hugo new post/first_post.md,之后在content/post目录会生成一个MarkDown格式的first_post.md文件。 获取主题 还是在YOUR_LOCAL_DIR目录下，创建一个存放主题的themes目录mkdir themes,进入themes目录，下载一个你喜欢的主题 git clone https://github.com/jpescador/hugo-future-imperfect.git 主题皮肤列表（点这里）,开启本地调试模式(方便预览与调试) hugo server –theme=hugo-future-imperfect –buildDrafts –watch 编译并产生博客静态资源 hugo –theme=hugo-future-imperfect –baseUrl=“http://YOUR_GITHUB_ACCOUNT.github.io/" 这样目录会产生public目录和下面各种js、css、html静态资源，这些就是我们博客网站展现的资源，即后面要上传到github的东东。 提交至Github 好了，终于到了github登场了。在github上创建一个存放你的博客内容的仓库,取名为YOUR_GITHUB_ACCOUNT.github.io,进入public目录，执行如下git命令： git init git remote add origin https://github.com/YOUR_GITHUB_ACCOUNT/YOUR_GITHUB_ACCOUNT.github.io.git git add -A git commit -m "first commit" git push -u origin master 嗯,到此你就可以访问http://YOUR_GITHUB_ACCOUNT.github.io.
