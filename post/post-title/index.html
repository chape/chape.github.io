这是我的第一篇博客,先尿个尿标记下如何使用git pages创建这个个人博客吧，^_^

操作环境：
      mac OSX
工具：
      iTerm
      git
      Hugo
前期准备：
      github账户
      brew包管理
开始：
第一步，使用brew安装hugo,brew install hugo,然后使用hugo命令hugo new site YOUR_LOCAL_DIR,运行后YOUR_LOCAL_DIR下会产生如下目录结构：
    archetypes
    content
    layouts
    static
    config.toml

archetypes目录还母鸡这个是用来干嘛的，和maven脚手架应该差不多，简化创建过程用的吧。content目录存放markdown文章，layouts目录存放网站的布局模板文件，static目录里放的是一些图片、css、js等资源。

进入YOUR_LOCAL_DIR目录,cd YOUR_LOCAL_DIR，创建博客MD文件，hugo new post/first_post.md,之后在content/post目录会生成一个MarkDown格式的first_post.md文件。 还是在YOUR_LOCAL_DIR目录下，创建一个存放主题的themes目录mkdir themes,进入themes目录，下载一个你喜欢的主题git clone https://github.com/jpescador/hugo-future-imperfect.git,主题皮肤列表（点这里）,开启本地调试模式hugo server –theme=hugo-future-imperfect –buildDrafts –watch，然后在运行hugo –theme=hugo-future-imperfect –baseUrl=“http://YOUR_GITHUB_ACCOUNT.github.io/",这样目录会产生public目录和下面各种js、css、html静态资源，这些就是我们博客网站展现的资源，即后面要上传到github的东东。
好了，终于到了github登场了。在github上创建一个存放你的博客内容的仓库,取名为YOUR_GITHUB_ACCOUNT.github.io,进入public目录，执行如下git命令：
git init
git remote add origin https://github.com/YOUR_GITHUB_ACCOUNT/YOUR_GITHUB_ACCOUNT.github.io.git
git add -A
git commit -m “first commit”
git push -u origin master. 好了,到此你就可以访问http://YOUR_GITHUB_ACCOUNT.github.io. 如果想添加新的博客，则重复在YOUR_LOCAL_DIR目录运行hugo new post/second_post.md
hugo –theme=hugo-future-imperfect –baseUrl=“http://chape.github.io/"
然后进入public目录的git repo提交修改到github即可。打完收工！
